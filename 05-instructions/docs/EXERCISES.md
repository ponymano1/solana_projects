# 练习题

## 基础练习

### 练习1：添加优先级字段

为Todo项添加优先级字段（Low, Medium, High）。

**要求：**
1. 在TodoItem中添加priority字段（使用枚举）
2. 在CreateTodo指令中添加priority参数
3. 添加一个新指令UpdatePriority来修改优先级
4. 编写测试验证功能

**提示：**
```rust
#[derive(BorshSerialize, BorshDeserialize, Debug, Clone)]
pub enum Priority {
    Low,
    Medium,
    High,
}
```

### 练习2：添加截止日期

为Todo项添加截止日期功能。

**要求：**
1. 在TodoItem中添加due_date字段（使用Option<i64>存储Unix时间戳）
2. 修改CreateTodo指令支持可选的截止日期
3. 添加一个新指令SetDueDate来设置/修改截止日期
4. 编写测试验证功能

**提示：**
- 使用Option<i64>表示可选的时间戳
- 考虑如何验证时间戳的有效性

### 练习3：批量操作

实现批量标记Todo为完成的功能。

**要求：**
1. 添加一个新指令BatchUpdateTodo，接受Vec<u32>作为ID列表
2. 实现批量更新逻辑
3. 处理部分ID不存在的情况
4. 编写测试验证功能

**提示：**
- 考虑如何处理错误：全部失败还是部分成功？
- 注意Vec的序列化大小限制

## 进阶练习

### 练习4：添加标签系统

为Todo项添加标签（tags）功能。

**要求：**
1. 在TodoItem中添加tags字段（Vec<String>）
2. 限制每个Todo最多5个标签，每个标签最多20个字符
3. 添加AddTag和RemoveTag指令
4. 实现标签去重逻辑
5. 编写完整的测试

**挑战：**
- 如何高效地查找包含特定标签的Todo？
- 如何防止标签重复？

### 练习5：实现软删除

将DeleteTodo改为软删除（标记为已删除而不是真正删除）。

**要求：**
1. 在TodoItem中添加is_deleted字段
2. 修改DeleteTodo指令实现软删除
3. 添加PermanentDelete指令实现真正的删除
4. 修改其他操作跳过已删除的Todo
5. 添加Restore指令恢复已删除的Todo
6. 编写完整的测试

**思考：**
- 软删除的优缺点是什么？
- 如何清理长期未使用的已删除Todo？

### 练习6：添加搜索功能

实现按标题搜索Todo的功能。

**要求：**
1. 添加SearchTodo指令，接受搜索关键词
2. 返回匹配的Todo ID列表
3. 支持大小写不敏感搜索
4. 考虑如何返回搜索结果（提示：可能需要修改程序返回值的方式）

**挑战：**
- Solana程序如何返回数据给客户端？
- 如何处理搜索结果过多的情况？

## 高级练习

### 练习7：实现权限系统

添加多用户支持和权限管理。

**要求：**
1. 修改TodoList支持多个用户
2. 添加角色系统（Owner, Editor, Viewer）
3. 实现AddCollaborator和RemoveCollaborator指令
4. 根据角色限制操作权限
5. 编写完整的测试

**设计考虑：**
- 如何存储多个用户的权限？
- 如何高效地检查权限？
- 账户空间如何计算？

### 练习8：实现Todo分类

添加Todo分类（Category）功能。

**要求：**
1. 创建Category结构存储分类信息
2. 每个TodoList可以有多个分类
3. 每个Todo属于一个分类
4. 实现CreateCategory, UpdateCategory, DeleteCategory指令
5. 实现MoveTodoToCategory指令
6. 编写完整的测试

**挑战：**
- 如何处理分类被删除时的Todo？
- 如何限制分类的数量？

### 练习9：添加统计功能

实现Todo统计信息。

**要求：**
1. 在TodoList中添加统计字段（总数、完成数、未完成数）
2. 在每次操作后更新统计信息
3. 添加GetStatistics指令返回统计数据
4. 确保统计信息始终准确
5. 编写测试验证统计的准确性

**思考：**
- 实时计算 vs 缓存统计，哪个更好？
- 如何确保统计信息不会出错？

### 练习10：实现数据迁移

实现从旧版本数据结构迁移到新版本的功能。

**场景：**
假设你需要在TodoItem中添加一个新字段created_at（创建时间）。

**要求：**
1. 定义新的TodoItemV2结构
2. 实现Migrate指令将V1数据转换为V2
3. 保持向后兼容性
4. 编写测试验证迁移过程

**挑战：**
- 如何识别数据版本？
- 如何确保迁移的原子性？
- 如何处理迁移失败的情况？
