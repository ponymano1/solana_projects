```rust

    is_initialized: bool,      // 1字节
    owner: Pubkey,             // 32字节
    name: [u8; 32],           // 32字节
    name_len: u8,             // 1字节
    age: u8,                  // 1字节
```

## 技术亮点

### 1. 固定大小数据布局
使用固定大小数组而非动态String，确保：
- 空间可预测
- 序列化高效
- 租金计算简单

### 2. 完整的生命周期管理
- 创建：System Program + 初始化
- 使用：读取、更新、验证
- 关闭：返还租金、清空数据

### 3. 安全检查
- 签名者验证
- 所有权验证
- 初始化状态检查
- 数据长度验证

### 4. 清晰的权限模型
- Account.owner：程序控制账户
- UserProfile.owner：用户控制数据

## 学习要点

### 核心概念
1. **数据存储**：Account.data字段
2. **两层Owner**：账户owner vs 数据owner
3. **租金机制**：防止状态膨胀
4. **序列化**：Borsh高效二进制格式

### 最佳实践
1. 使用固定大小数据结构
2. 验证所有输入

5. 提供清晰的错误消息

## 与以太坊对比

| 特性 | Solana | 以太坊 |
|------|--------|--------|
| 存储模型 | 独立账户 | 合约内部 |

| 数据访问 | 显式传递 | 隐式访问 |
